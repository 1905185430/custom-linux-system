# GRUB 启动项自动生成脚本

## 适用环境

| 项目 | 说明 |
|------|------|
| 操作系统 | Ubuntu 22.04 LTS（或其他基于 Debian 的发行版） |
| 引导程序 | GRUB 2 |
| 运行场景 | VMware / QEMU 虚拟机，或实体机 |

---

## 功能说明

`scripts/add-all-initrd-grub.sh` 会：

1. 扫描 `/boot/initrd.img-<version>` 文件（跳过无版本号的别名，如 `initrd.img`、`initrd.img.old`）。
2. 对每个版本，查找对应的 `/boot/vmlinuz-<version>`；若不存在则跳过并打印警告。
3. 在 `/etc/grub.d/40_custom` 中用 `### BEGIN / END AUTO-GENERATED INITRD ENTRIES` 标记块管理自动生成的启动项——每次运行先删除旧块再写入新块，手动编写的其他条目不受影响。
4. 自动执行 `update-grub`；若不存在则回退到 `grub-mkconfig -o /boot/grub/grub.cfg`。

---

## 安装与运行

### 基本用法

```bash
sudo bash scripts/add-all-initrd-grub.sh
```

### 带自定义内核参数

```bash
sudo KERNEL_ARGS="quiet splash" bash scripts/add-all-initrd-grub.sh
```

### 自定义菜单标题前缀

```bash
sudo TITLE_PREFIX="MyLinux" bash scripts/add-all-initrd-grub.sh
# 生成的条目标题示例：MyLinux (kernel 6.8.0-90-generic)
```

### 组合使用环境变量

```bash
sudo TITLE_PREFIX="Custom OS" \
     KERNEL_ARGS="root=/dev/ram0 rw console=ttyS0" \
     bash scripts/add-all-initrd-grub.sh
```

---

## 环境变量参考

| 变量 | 默认值 | 说明 |
|------|--------|------|
| `BOOT_DIR` | `/boot` | 扫描 initrd/vmlinuz 的目录 |
| `GRUB_CUSTOM` | `/etc/grub.d/40_custom` | 写入 menuentry 的目标文件 |
| `TITLE_PREFIX` | `Custom Linux` | GRUB 菜单条目的标题前缀 |
| `KERNEL_ARGS` | _(空)_ | 附加到 `linux` 行的内核参数 |

## 脚本行为详解

### 文件修改

脚本会修改 `/etc/grub.d/40_custom`，具体规则：

- 若文件不存在，自动创建并写入 GRUB 所需的 shebang 头部。
- 仅替换 `### BEGIN AUTO-GENERATED INITRD ENTRIES` … `### END AUTO-GENERATED INITRD ENTRIES` 标记块内的内容；文件中其他手写条目**不受影响**。

### 生成的 menuentry 示例

```
### BEGIN AUTO-GENERATED INITRD ENTRIES
# Generated by /path/to/add-all-initrd-grub.sh on 2024-01-01T00:00:00+00:00
# Do not edit this block manually – it will be overwritten on next run.

menuentry "Custom Linux (kernel 6.8.0-90-generic)" {
    linux   /boot/vmlinuz-6.8.0-90-generic quiet splash
    initrd  /boot/initrd.img-6.8.0-90-generic
}

menuentry "Custom Linux (kernel 5.15.0-100-generic)" {
    linux   /boot/vmlinuz-5.15.0-100-generic quiet splash
    initrd  /boot/initrd.img-5.15.0-100-generic
}

### END AUTO-GENERATED INITRD ENTRIES
```

### GRUB 更新

脚本最后调用 `update-grub`（Ubuntu 22.04 默认可用），若不存在则自动回退：

```
grub-mkconfig -o /boot/grub/grub.cfg
```

---

## 注意事项

- **需要 root 权限**：脚本开头会检测，若非 root 则立即退出并提示使用 `sudo`。
- **无需额外安装包**：仅依赖系统自带的 `bash`、`awk`、`mktemp`、`update-grub` / `grub-mkconfig`。
- **不依赖网络**：所有操作均在本地完成。
- **跳过别名文件**：`initrd.img`、`initrd.img.old`、`vmlinuz`、`vmlinuz.old` 等无版本号的文件会被自动跳过。
